### 📝 变更内容
新增适用于当前 Yuxi 项目的 Conda 宿主机开发环境手册，记录 Python 3.13、uv、Node.js 24 和 pnpm 10.11.0 的职责边界、环境创建、依赖安装、配置初始化、四服务启动、测试、常见问题、更新重建和验收清单；同时在拆分部署 README 中增加该手册索引。

### 🔍 变更原因
本机 Conda base 环境缺少 uv，且现有 Node.js 版本低于项目要求，需要形成一套与现有 uv/pnpm 锁文件和宿主机启动脚本兼容的可复用环境创建流程。

### ⚠️ 注意事项
本次仅调整说明文档，不改变程序运行行为。推荐方案使用 Conda 隔离 Python、uv 和 Node 工具版本，但后端真实业务依赖仍由 `backend/uv.lock` 安装到 `backend/.venv`。Docker daemon 不由 Conda 管理，使用本地 Sandbox 时仍需单独安装。`docs/vibe` 当前受 `.gitignore` 忽略，需要团队追踪时应明确调整文档版本控制策略。

### 🧪 测试建议
已使用 Conda 25.9.1 对 `python=3.13 uv nodejs=24` 执行创建环境的 dry-run 验证，并检查文档引用文件、Markdown 代码块和项目版本依据。实际验收应新建 `yuxi-dev` 环境，执行依赖安装脚本，并验证 API、Worker、Web 与 Sandbox Provisioner 启动。
