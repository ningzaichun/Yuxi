### 📌 变更类型
修改 / 配置

### 📁 影响范围
- `backend/server/routers/system_router.py`
- `backend/test/unit/routers/test_system_router.py`
- `web/src/stores/info.js`
- `docs/develop-guides/changelog.md`
- 系统品牌配置接口与前端品牌信息 Store

### 📝 变更内容
默认品牌配置文件改为基于后端源码目录定位，不再依赖 API 进程的当前工作目录，同时保留 `YUXI_BRAND_FILE_PATH` 自定义路径。前端组织信息增加量程科技名称、正式 Logo、头像与登录背景默认值，品牌接口暂时不可用时不再显示破图。新增单元测试验证从任意工作目录启动时仍可读取品牌模板。

### 🔍 变更原因
宿主机启动脚本从仓库根目录运行 API，而品牌接口此前使用 `package/yuxi/...` 工作目录相对路径，导致配置文件读取失败并返回空对象，首页得到空 Logo 地址后显示破图。

### ⚠️ 注意事项
显式配置的 `YUXI_BRAND_FILE_PATH` 仍优先于默认品牌文件。当前机器另有项目占用 `127.0.0.1:5173`，Yuxi 开发页面应通过 `http://localhost:5173` 访问，或先停止其他项目后再统一端口。

### 🧪 测试建议
- 运行 `backend/.venv/Scripts/python.exe -m pytest test/unit/routers/test_system_router.py`。
- 请求 `/api/system/info`，确认返回 `organization.logo=/k-ai-logo.png`。
- 运行 `pnpm --dir web run build` 并检查首页 Logo。
