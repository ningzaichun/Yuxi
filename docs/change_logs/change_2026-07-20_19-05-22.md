### 📝 变更内容
删除 Infra/App 的 PowerShell Compose 包装脚本、内网 IP 专用环境模板、旧项目分析、历史逐次日志和已被替代的日期化长手册。宿主机启动脚本收敛为 API、Worker、Web；本地 Sandbox Provisioner 改由开发 Compose 单独启动，并通过绝对线程目录与宿主机程序共享文件。补充精简的本地开发指南、直接 Docker Compose 部署命令和当前项目分析报告，并同步纠正自动化助手的开发/测试环境说明。

### 🔍 变更原因
当前开发和生产拓扑已经明确，原仓库仍同时展示多套过期入口与重复说明，增加了部署选择和阅读成本。本次清理保留一个开发拓扑及一体化、拆分两种生产拓扑，为后续业务开发建立稳定入口。

### ⚠️ 注意事项
本次不修改业务源码、数据库结构或 API 行为，也不停止现有容器。首次切换到 Dockerized Provisioner 前，应停止本机由生产 Compose 启动的同名 Provisioner，并按本地开发指南设置 `SANDBOX_DOCKER_THREADS_HOST_PATH`，避免端口、容器名和文件目录冲突。日常 Compose 停止命令禁止使用 `--volumes`。
