### 📝 变更内容
新增独立的 `get_weather` 内置工具，通过 Open-Meteo 地理编码和当前天气接口按城市查询天气，返回规范城市、天气描述、温度、体感温度、湿度、风速和观测时间。同步补充入口注册、成功与异常场景单元测试及正式变更记录。

### 🔍 变更原因
验证内置工具按独立文件组织、后端自动注册和前端通用展示的扩展路径，并为智能体提供无需 API Key 的实时天气查询能力。

### ⚠️ 注意事项
首版依赖运行环境能够访问 Open-Meteo；不包含未来预报、空气质量、历史天气、同名城市交互选择、缓存、重试或其他天气供应商回退。现有已保存为显式工具列表的智能体需要勾选 `get_weather` 后才能调用。

### 🧪 测试建议
- 运行天气工具单元测试，验证正常查询、城市不存在、响应不完整、请求失败和注册元数据。
- 运行工具注册与元数据服务回归测试。
- 验证工具元数据中包含 `get_weather`，并在开发环境执行一次真实城市查询。
