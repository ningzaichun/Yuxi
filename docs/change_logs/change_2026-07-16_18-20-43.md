### 📝 变更内容
将 Conda 手册从“Conda 同时管理 Python、uv 和 Node.js”调整为当前机器适用的分工：Conda 只管理 Python 3.13 与 uv，现有 NVM for Windows 管理 Node.js 24，pnpm 独立管理。同步更新环境创建命令、命令来源检查、每日启动、Node 版本排障、环境更新、验收清单和结论，并记录当前 NVM、Node 与 pnpm 的实际版本和路径。

### 🔍 变更原因
当前机器已经使用 NVM 管理 Node.js。继续在 Conda 环境中安装 Node.js 会让两个版本管理器同时向 PATH 提供 `node.exe`，增加版本漂移和排障成本。

### ⚠️ 注意事项
本次仅修改说明文档，不改变程序行为。当前推荐每个开发终端先激活 `yuxi-dev`，再执行 `nvm use 24.0.0`。pnpm 当前为 10.33.4，项目声明为 10.11.0；通常同属 pnpm 10 可以工作，严格复现时仍应固定到项目声明版本。`docs/vibe` 当前受 `.gitignore` 忽略。

### 🧪 测试建议
已检查当前 `nvm current` 为 24.0.0、`node.exe` 来自 `D:\dev\env\nodejs`、pnpm 为 10.33.4，并校验文档不存在生效中的 Conda Node 安装步骤。实际使用时应创建 `yuxi-dev`，验证 Conda 的 Python/uv 与 NVM 的 Node 能在同一终端正确解析。
