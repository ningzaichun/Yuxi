# 建筑领域知识图谱演示数据

本目录提供一套围绕虚构项目“滨江科创中心一期工程”的知识图谱测试材料，用于验证 Yuxi 的文档解析、实体关系抽取、Neo4j 图谱构建和图增强检索链路。

## 文件用途

| 文件 | 是否上传 | 内容 |
| --- | --- | --- |
| `01-project-overview.md` | 是 | 项目概况、建设地点、单体和建设单位 |
| `02-participants.md` | 是 | 参建单位、人员及其角色 |
| `03-buildings-and-technologies.md` | 是 | 单体结构、施工技术、材料和设备 |
| `04-contracts-and-suppliers.md` | 是 | 合同、供应商、材料和设备供应关系 |
| `05-quality-and-safety-events.md` | 是 | 质量安全事件、影响范围和整改责任 |
| `06-progress-and-changes.md` | 是 | 进度节点、设计变更和整改闭环 |
| `expected-entities.csv` | 否 | 人工整理的核心实体基线 |
| `expected-relations.csv` | 否 | 人工整理的核心关系基线 |
| `validation-questions.md` | 否 | 单文档、跨文档和无答案验证问题 |

## 推荐用法

1. 仅上传编号 `01` 至 `06` 的 Markdown 文件，不要上传本说明、CSV 和问题清单。
2. 首次试建只上传 `01`、`02`、`03`，检查 Schema 和实体归一化效果。
3. 调整合格后上传 `04`、`05`、`06`，对待处理 Chunk 执行增量构建。
4. 将图谱界面和检索结果与两个 CSV 及验证问题对照。

标准答案只用于人工评审。LLM 抽取具有非确定性，实体和关系数量不要求与 CSV 完全相等，但核心关系应有原文证据且方向正确。
